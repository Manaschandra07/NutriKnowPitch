01. THE PROBLEM

Getting fit is a global goal, but the food industry has rigged the game against you.

Supermarket aisles are filled with products claiming to be "all-natural" or "sugar-free," but the reality is much darker. The modern food ecosystem is engineered to confuse consumers and hide cheap, inflammatory ingredients.

Time & Friction

People are busy, and deciphering modern nutrition labels practically requires a chemistry degree. Constantly analyzing INS numbers and complex ingredients drains energy. Because of this friction, human laziness usually wins out.

Regulatory Loopholes

Food companies exploit information asymmetry. A label boasting "Zero Sugar" legally hides highly processed Maltodextrin—an additive that spikes blood glucose faster than table sugar. Consumers are fundamentally deceived by legal "greenwashing."

The Metabolic Toll

The reliance on ultra-processed foods (UPFs) loaded with inflammatory seed oils (Palm, Vanaspati) and synthetic dyes creates a cocktail effect. We blindly ingest ingredients directly linked to long-term metabolic dysfunction because the packaging looks "healthy."

NutriKnow Architecture

Under the Hood

NutriKnow is engineered as a seamless, client-side heavy React application. It utilizes modern Web APIs for hardware access, proprietary string-matching arrays for ingredient decryption, and connects to a backend AI engine to parse complex label imagery. Here is the technical breakdown of how data flows from the user to the backend.

PART 1: THE USER EXPERIENCE

How to Use NutriKnow

1. Data Input (The Scan)

NutriKnow is designed to be completely frictionless. Users are presented with three input methods:

  • Live Camera Scan: The user points their smartphone camera at the product's barcode. The app instantly recognizes the format.
  • Manual Entry: Users can manually type the 8 or 13-digit barcode.
  • Label OCR (Image Upload): If the product isn't in any database, the user snaps a photo of the ingredient list and nutrition panel.
2. Data Verification

If the user uploads an image, the app's AI extracts the text. Before analysis, the user is shown a "Verification Screen" where they can review and correct any numbers the AI might have misread, ensuring 100% accuracy.

3. The Intelligence Dashboard

Once data is fetched, the core analysis is presented via several modular UI components:

  • The Arc Dial: A visual "Nutri-Score" out of 100, encircled by rings for Fiber, Fat, and Sugar.
  • How They Are Fooling You: Flags "Greenwashing" (e.g., claiming "Multigrain" when fiber is < 3g).
  • Ingredient Decryption Engine: Translates chemical names into English, highlighting hidden sugar aliases in red.
  • Nutritional Karma Ledger: Total transparency into how the score was calculated (points deducted for sodium, added for protein).
  • Ancient Wisdom Swaps: Suggests healthier, traditional alternatives based on Ayurveda and modern science.

PART 2: TECHNICAL ARCHITECTURE

Under the Hood

1. Data Ingestion & API Routing
  • Barcode Detection: Utilizes the native window.BarcodeDetector Web API for hardware-accelerated scanning directly through the browser.
  • Open Food Facts Integration: Pings the open-source Open Food Facts API. The app includes a dedicated normalization function to clean irregular JSON data, standardizing macros to strict "per 100g" formats.
  • Vision AI Pipeline: For image uploads, images are converted to Base64 and routed securely to an internal endpoint. This endpoint communicates with a multimodal LLM (Claude Sonnet 4.6), strictly prompted to return clean JSON overriding markdown.
2. The Decryption Engine

Ingredient arrays run through proprietary databases sorted "longest-name-first" to ensure exact substring matching without duplication:

  • Sugar & Oil DBs: Cross-references against lists of covert ingredients (e.g., Maltodextrin, Vanaspati).
  • Additive Regex: Uses Regex /INS\s*\d+|E\d{3,4}/i to flag artificial colorings and emulsifiers instantly.
  • Greenwashing Algorithm: Compares marketing claims against actual macros (e.g., claiming "Sugar Free" while addedSugar > 0.5g triggers a flag).
3. Bounded Scoring Algorithm

The core IP calculates the score using a strictly controlled mathematical formula where every variable is hard-capped to prevent skewed data:

Score = max(10, min(100, 100 - Penalties + Bonuses))
  • Penalties: Added sugars (capped at 30 pts), bad oils (capped at 25), sodium (capped at 15), additives (capped at 20).
  • Bonuses: Dietary fiber (capped at 10 pts), protein (capped at 10), Ancient Wisdom ingredients (capped at 15).

Because the algorithm calculates exact integers for penalties/bonuses, those exact integers are pushed to the UI's "Karma Ledger", ensuring mathematical transparency.

4. Context-Aware RAG Chatbot

When the user opens the AI assistant, the application dynamically constructs a highly specific system prompt. It injects the product's name, brand, computed health score, the exact formula used to grade it, identified sugar aliases, harmful oils, and allergens. This ensures the LLM is tightly grounded in the factual data of the scanned product, preventing hallucinations and ensuring scientifically accurate dietary advice.
